## 一、 计算机系统基础结构
### 1. 冯·诺依曼架构 (Von Neumann Architecture)
- 基本原理：程序和数据均存储在内存中。
- 五大组成部分：
    1.  **输入 (Input)**
    2.  **输出 (Output)**
    3.  **存储器 (Memory)**
    4.  **控制器 (Control)**：指挥数据通路、内存和 I/O 设备。
    5.  **数据通路 (Datapath)**：执行算术运算。
- CPU 运作机制：从内存中提取指令（Fetch），解码并执行，最后将结果写回。



### 2. 软硬件接口：指令集架构 (ISA)
**ISA (Instruction Set Architecture)**：是软件和硬件之间的接口。
- **软件视角**：ISA 提供了编程所见的机器抽象（如寄存器、指令集）。
- **硬件视角**：ISA 是硬件必须实现的规范。

## 二、 性能定量分析 (Quantitative Principles of Performance)
### 1. 性能的定义
**性能 (Performance) = 1 / 执行时间 (Execution Time)**。
**响应时间 (Response Time/Latency)**：从任务开始到完成的时间。
**吞吐量 (Throughput)**：单位时间内完成的总任务量。

### 2. CPU 性能公式 (The Classic CPU Performance Equation)
> $$\text{CPU Time} = \text{Instruction Count (IC)} \times \text{CPI} \times \text{Clock Cycle Time } (T_c)$$

- **IC (指令数)**：由算法、程序设计、编译器和 ISA 决定。
- **CPI (每条指令周期数)**：由硬件实现（微架构）和 ISA 决定。
- **Clock Cycle Time ($T_c$)**：由硬件工艺和流水线设计决定。



### 3. 阿姆达尔定律 (Amdahl’s Law)
**核心定义**：对系统中某一部分进行改进所获得的性能提升，受限于该部分被使用的频率。

> $$\text{Speedup}_{\text{overall}} = \frac{1}{(1 - \text{Fraction}_{\text{enhanced}}) + \frac{\text{Fraction}_{\text{enhanced}}}{\text{Speedup}_{\text{enhanced}}}}$$

**结论**：加速比受限于“无法增强的部分”。
**推论**：应当“加速经常性事件 (Make the common case fast)”。



## 三、 计算机架构的八大思想 (8 Architecture Ideas)
1.  **面向摩尔定律设计**：预测未来工艺。
2.  **使用抽象简化设计**：分层管理复杂性。
3.  **加速经常性事件**：性能优化的重点。
4.  **通过并行提高性能**：如多核、向量处理。
5.  **通过流水线提高性能**：重叠执行指令。
6.  **通过预测提高性能**：如分支预测。
7.  **存储器层次结构**：利用局部性原理（Cache）。
8.  **通过冗余提高可靠性**：容错处理。
